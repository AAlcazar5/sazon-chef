# DS10.1 — Image fallback strategy

> **Decision:** Recipe images that fail to load fall back through a
> two-stage chain: blur-up placeholder → mascot-on-pastel scrim. Tokens
> are codified in `ImageState` (DS3.5).

## Fallback chain

| Stage | Trigger | Render |
|------:|---------|--------|
| 0 | `src` is loading | `ImageState.placeholder.bg.{light,dark}` solid fill (warm-gray idle, no shimmer to avoid competing with cards). |
| 1 | `src` is undefined / null | `ImageState.fallback.mascot = 'curious'` mascot centered on `Pastel.peach` (light) / pastel-dark peach (dark). |
| 2 | `src` set but `Image` reports an error | `ImageState.error.bg.{light,dark}` (red wash) + `ImageState.error.mascot = 'thinking'` (NEVER `sad` — voice rule, see DS4.7). |

## Implementation contract

A reusable `<RecipeImage>` (or `<SafeImage>`) primitive consumes the
`ImageState` tokens directly. Hand-rolled fallbacks in screens are
banned — always reach for the primitive.

Skeleton-style shimmer on stage 0 is intentionally avoided. Skeleton
loaders (DS3.4) are for whole-card placeholders; an image inside a
loaded card uses a calmer solid placeholder so the card doesn't feel
busy.

## Why no third-party CDN fallback

Sazon ships with curated photography for the seeded corpus. AI-generated
images from `recipeImageService` are also self-hosted. There is no
external image source in the cold path, so a generic CDN-fallback layer
is unnecessary complexity.

## Token source

`frontend/constants/colorTokens.cjs` → `ImageState`.
`frontend/constants/tokens.ts` → re-export.

Component wire-up (a `<RecipeImage>` primitive consuming the chain) is
the DS7-tier follow-up.
