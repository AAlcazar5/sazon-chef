# DS10.2 — Mascot expression matrix

> **Decision:** Every UI state maps to exactly one Sazon expression.
> The pairing rule lives in `constants/MascotForState.ts` (DS4.7); this
> doc is the editorial / design-side mirror so the rule appears
> alongside visual specs.

## State → expression matrix

| State              | Expression    | When it shows                                            | Voice |
|--------------------|---------------|-----------------------------------------------------------|-------|
| `loading`          | `thinking`    | Any data fetch in flight                                  | "Sazon's pulling this together for you…" |
| `success`          | `chef-kiss`   | Save / publish / completed milestone                      | "Locked in." / "Beautiful." |
| `cooking-complete` | `chef-kiss`   | Cook timer hits zero, recipe-complete sheet               | "Magnifico." |
| `search-empty`     | `curious`     | Search query yields no results, but the box has text      | "Hmm — let me think about that one." |
| `no-notifications` | `sleepy`      | Notifications inbox empty                                 | "All quiet on the kitchen front." |
| `no-results`       | `curious`     | Filter / cuisine / craving returns empty                  | "No matches there — want to widen the net?" |
| `idle`             | `idle`        | Generic empty surface, no specific state                  | "Tap me when you need something." |
| `error`            | `thinking`    | Any user-facing failure — never `sad` / `angry`           | "Hmm — that didn't go through. Try once more." |
| `streak`           | `celebrating` | Milestone hit (e.g., 7-day cooking streak)                | "Look at you — that's three weeks straight." |
| `milestone`        | `celebrating` | First save, 10th cook, badge unlock                       | "First one in the books." |
| `first-launch`     | `happy`       | Welcome / onboarding step 1                               | "Hey, glad you're here." |

## Anti-patterns

- **Sad / angry / frustrated mascot expressions are banned.** Errors
  surface as a curious / thinking moment, not a punitive one. `Sazon`
  is the brand-as-friend, never the brand-as-judge.
- **No mascot for purely chrome states** (tab bar, header, scroll
  indicator). The mascot is reserved for moments that earn personality
  attention — loading, empty, success, error.

## Test enforcement

`frontend/__tests__/constants/MascotForState.test.ts` (5/5):
- Every state in the enum has a non-empty expression.
- `error` maps to `thinking`, never `sad`/`angry`.
- `cooking-complete` always lands on `chef-kiss` (the joy peak).
- `BANNED_EXPRESSIONS` (`sad / angry / frustrated / crying`) absent
  from every mapping.
- `expressionFor()` helper returns the same value as the map lookup.
