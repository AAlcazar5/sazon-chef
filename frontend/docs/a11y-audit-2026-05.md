## Known violation (documented, not silenced)

**`Brand.light.base` (#fa7e12) used as a solid CTA background fails WCAG AA 3:1 with white text.**

- Affected: `BrandButton` (`components/ui/BrandButton.tsx`) when rendered with the `brand` variant on a light theme. The button uses a *gradient* from `Brand.light.base` â†’ `Semantic.light.error` (#EF4444), so the visual mean is darker than the start point â€” but the WCAG audit measures the worst case.
- Workaround for new surfaces: use `Brand.light.deep` (#d67a0c, 3.17:1) for solid-color CTA backgrounds. The audit asserts this pair instead.
- Product decision needed for the existing `brand` variant gradient: (a) darken the gradient start to `Brand.light.deep` (would change visual brand identity), (b) keep the gradient + accept the violation as a known compromise, or (c) raise the text to a slightly off-white that clears 3:1 while preserving the visual feel. Tracked but not auto-fixed.
