# Codebase audit — 2026-05-11 (post-Tier-U)

Tier U closed 12/12. A fresh pass across perf, security, DX, observability,
i18n, and test coverage surfaced the following gaps NOT covered by U.
Findings are graded by **launch impact**, not severity in the abstract.

---

## Critical — fix before TestFlight

### C1. Hardcoded JWT_SECRET fallback string (3 sites)

```
backend/src/modules/auth/socialAuthController.ts:10
backend/src/modules/auth/authController.ts:23
backend/src/services/refreshTokenService.ts:21
```

All three contain:

```ts
const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key-change-in-production';
```

The `authMiddleware.ts` startup guard refuses to boot without a 32+ char
`JWT_SECRET`, so the fallback is currently dead. **But** — if anyone
removes that guard (cleanup pass, env-loading refactor), the fallback
becomes live and tokens are forgeable. Delete the fallback in all three
files; let the guard be the single source of truth.

**Fix:** `const JWT_SECRET = process.env.JWT_SECRET!;` (non-null assertion
because the startup guard enforces it).

### C2. O(N) email scan on every login / register (real scaling cliff)

```
backend/src/modules/auth/authController.ts  (5 findMany sites)
```

Encrypted emails can't be queried directly. The current flow:

```ts
const usersWithEncryptedEmails = await prisma.user.findMany({
  where: { emailEncrypted: true },
  select: { id: true, email: true, emailEncrypted: true },
});
// then decrypt-and-compare in a loop until found
```

At 1k users this is fine. At 100k it's 100k decrypts on every failed
login. Standard mitigation: add an `emailLookupHash` column (HMAC-SHA256
of normalized email with a server-side key, deterministic, unique).
Lookup becomes O(1). Keep `email` encrypted for display.

**Cost:** schema migration + backfill. Worth scoping a Tier-V item.

## High — fix before public launch

### H1. `Alert.alert(...)` strings (161 sites, mostly bypassing i18n)

Tier-Q M5 i18n pass covered headline copy. 161 raw `Alert.alert('Oops!',
'...')` calls remain — most live in `app/*.tsx` (`build-a-plate.tsx`,
`scanner.tsx`, `recipe-form.tsx`, etc.). Many use English-only strings
that won't translate.

**Fix:** systematic sweep via codemod (`Alert.alert(t('foo.title'),
t('foo.body'))`) + lint rule against literal strings in `Alert.alert`.

### H2. TODO placeholders that will ship as user-facing data

```
frontend/hooks/useMealPlanActions.ts:1007  calories: 500,           // TODO: Get from recipe data
frontend/hooks/useMealPlanActions.ts:1011  description: "Delicious recipe added to your meal plan",
frontend/hooks/useMealPlanActions.ts:1012  prepTime: "15 min",
frontend/hooks/useMealPlanActions.ts:1013  difficulty: "Easy"
```

Hardcoded `500 cal` + `"15 min"` on the meal-plan add flow means users
see incorrect nutrition during the optimistic state. Either pull from
the recipe (right answer) or drop the keys and let the cache fill them.

```

frontend/app/(tabs)/meal-plan.tsx                     1365 lines
```

CLAUDE.md sets 800-line max as "touch hazard". These 10 files are 1.7×
to 7× over. Refactoring them ALL is a multi-week investment — but
`modal.tsx` (2668 lines) and `recipeController.ts` (5614 lines) are
genuine review-blocker territory. Scope: tier-by-tier split as features
land in those files, plus a banned-pattern lint that warns above 800.

### M2. Stale dependencies

```
@prisma/client       5.22.0  → 7.8.0   (2 majors behind)
@anthropic-ai/sdk    0.68.0  → 0.95.2  (model-routing API changes)
@sentry/node         10.40.0 → 10.52.0 (point upgrade — safe)
@types/express       4.17.23 → 5.0.6   (waiting on Express 5)
```

Prisma 5 → 7 is a deliberate post-launch task (breaking changes around
the client API). Anthropic SDK is closer to safe and adds model-version
flexibility — worth a controlled upgrade pre-launch.

### M3. Default helmet config (no explicit hardening)

`backend/src/app.ts:110` uses `app.use(helmet());` without overrides.
Helmet's defaults are OK for an API server, but explicit config makes
the security posture reviewable. Pin `contentSecurityPolicy`,
`hsts: { maxAge: 31536000, includeSubDomains: true, preload: true }`,
`referrerPolicy: { policy: 'no-referrer' }`. Same diff size as a comment
upgrade.

### M4. 18 unresolved TODO/FIXME markers

Most are non-blocking, but worth a sweep — file each into a Tier-V or
roadmap entry, or delete the comment if no longer relevant. Two H2
items above are already from this list.

### M5. setTimeout/setInterval without obvious paired clear (14 files)

```
app/login.tsx           set=5 clear=0
app/forgot-password.tsx set=6 clear=0
app/modal.tsx           set=4 clear=0
app/(tabs)/meal-plan.tsx set=2 clear=0
... 10 more
```

Many are likely setTimeout-once patterns that don't need clears
(animations, debouncing). But a 6-timer file with zero clears
(`forgot-password.tsx`) deserves a closer look — a leaked timer
firing setState on an unmounted component is a classic React warning
generator.
